## 一、项目背景与目标

### 1.1 科学问题

丝素蛋白（Silk fibroin）是蚕丝的主要蛋白成分，具有优异的自组装能力和成膜特性。我们的目标是：将具有特定生物活性的短肽（功能肽）与丝素蛋白融合，使重组蛋白兼具两者的优点——丝素蛋白提供结构支撑和递送能力，功能肽发挥抗氧化、美白等生物活性。

这类融合蛋白在医美外用场景（如护肤品、功能敷料）中有广阔的应用前景。

### 1.2 核心挑战

从生物序列数据库（如 UniProt 包含 2.25 亿条序列）中，筛选出最适合融合的功能肽候选，面临以下挑战：

- **候选规模巨大**：原始序列达数亿条，无法逐一实验验证
- **多维评估需求**：需同时考虑抗氧化活性、安全性（毒性、溶血性、致敏性）、结构稳定性、肽段暴露程度等
- **多指标权衡**：不同指标之间存在权衡关系，需要科学地综合评估

### 1.3 我们的方案

设计一套计算筛选Pipeline，通过多轮漏斗式筛选，从亿级候选逐步收敛至百级最优候选，再提交湿实验验证。

## 二、技术方案

### 2.1 Pipeline 架构：漏斗式多轮筛选

我们的筛选系统分为 8 个轮次（Round），每轮逐步收紧候选数量：

```
第 0 轮（数据预处理）：8.5 亿条 → 1990 万条
    ├── 长度过滤（3-30 个氨基酸）
    ├── 标准氨基酸过滤
    └── 去重

第 1 轮（抗氧化分选）：1990 万条 → ~200 万条
    ├── 抗氧化活性预测
    └── 双通道分选（Top 10% 抗氧化最好 + Bottom 1% 抗氧化最差作为阴性对照）

第 2 轮（安全筛检）：~200 万条 → ~150 万条
    ├── 毒性预测：≥ 0.38 淘汰
    ├── 溶血性预测：≥ 0.55 淘汰
    └── 免疫原性预测：≥ 0.50 淘汰

第 3 轮（深度评分）：~150 万条 → ~7.5 万条
    ├── B 细胞表位预测
    ├── 热稳定性预测
    ├── 溶解度预测
    ├── 细胞穿透肽预测
    └── 综合评分（方差驱动加权）

第 4 轮（构造枚举）：~7.5 万条 → ~2.25 万个融合构造
    ├── 融合位置枚举（N端 / C端 / 两端）
    ├── 连接子枚举（柔性 / 刚性）
    └── 组装全长序列

第 5 轮（3D 结构预测）：~2.25 万个 → 全量通过
    ├── OmegaFold 深度学习结构预测（主力，全量运行）
    └── AlphaFold3 高精度验证（仅 Top 10 + Bottom 10）

第 6 轮（结构评估）：~2.25 万个 → 全量评分
    ├── 溶剂可及表面积（SASA）：肽段暴露程度
    └── 聚集倾向分析：蛋白质聚集风险

第 7 轮（最终排名）：Top 100 + Bottom 100
```

**总体淘汰率**：> 99.99%（从 1990 万候选到 100 最终候选）

### 2.3 Docker 微服务架构

所有 16 个服务都封装在 Docker 容器中运行，带来以下好处：

- **环境一致性**：避免不同服务器配置差异导致的运行结果不一致
- **按需启动**：不需要同时运行所有服务，根据当前轮次只启动所需的 3-5 个服务，节约计算资源
- **GPU 资源管理**：部分服务（如 OmegaFold）需要 GPU，按需启动避免显存争抢

```bash
# 启动所有服务示例
cd tools && docker compose --profile gpu up -d          # GPU 服务
docker compose --profile cpu up -d                          # CPU 服务

# 按需启动单个服务
docker compose --profile cpu up -d anoxpepred algpred2
```

### 2.4 双通道设计

为确保筛选的科学严谨性，我们设计了双通道并行筛选：

- **Top 通道**：抗氧化活性最好的候选肽
- **Bottom 通道**：抗氧化活性最差的候选肽（作为阴性对照）

两组候选都经历相同的安全筛检和结构评估流程。最终两组各自独立排名，同时输出。这样可以验证：如果一个肽"抗氧化性差但其他指标都好"，它在阴性对照中也应该表现出相应的特征，从而排除假阳性。

## 三、技术难点与创新

### 3.1 方差感知权重：让数据决定哪个指标更重要

传统的加权评分方式是人工预设权重（如抗氧化 50%、毒性 13%、溶血 9% 等）。这样做的问题在于：不同指标的区分度在不同批次数据中是变化的——如果某个指标在当前候选池中大家都差不多（区分度差），给它再高的权重也没有意义。

**我们的创新**：先让所有候选跑完评分，再分析每个指标的分数分布（标准差），区分度好的指标自动获得更高权重。

```
标准差 σ_i 越大 → 该指标区分度越好 → 权重越高
```

这避免了"拍脑袋"定权重，让数据自己说话。

### 3.2 安全属性硬阈值：不是"可补偿"的评分项

在 stages2（旧版Pipeline）中，我们将安全性指标（毒性、溶血性、致敏性）以低权重混入综合评分。这意味着：一个溶血性极差的肽，只要抗氧化得分足够高，综合分仍然可能排在前列。

**我们发现的缺陷**：安全属性变成了"可以被抗氧化得分补偿"的评分项，这违背了药物安全性的基本原则。

**stages4（新版）的修复**：安全属性使用硬阈值——达标/淘汰，不参与任何评分混合。只要溶血性超过阈值，无论其他指标多优秀，直接淘汰。

### 3.3 抗氧化单次使用：避免双通道污染

在 stages2 中，虽然双通道分选时使用了纯抗氧化分数，但在后续轮次中，抗氧化权重仍然隐含在综合评分里，导致 Bottom 通道（抗氧化差）实际上还是被抗氧化残余权重影响。

**stages4 的修复**：抗氧化分数只在第 1 轮分选时使用一次，一旦通道确定，后续所有轮次不再使用抗氧化分数进行排名。双通道真正独立。

### 3.4 OmegaFold 阻塞问题与解决

OmegaFold 的深度学习推理是同步操作，会阻塞计算机的异步事件处理框架。我们通过信号量（Semaphore）控制并发数为 1，确保同一时间只有一个推理任务在运行，避免阻塞。

### 3.5 Docker 网络优化

早期我们通过本地端口（127.0.0.1）访问 Docker 容器中的服务，发现长耗时请求会间歇性挂死。改用 Docker 内部桥接网络 IP 直连后，问题解决。
